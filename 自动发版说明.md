# 自动发版说明

这个项目现在支持“提交修复后自动发 GitHub Release”。

## 安装

在项目根目录运行：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\tools\install_auto_release_hook.ps1 -RepoSlug ZhiKong0/review-baodian-apk
```

安装后会做两件事：

1. 把仓库的 `core.hooksPath` 指到版本化目录 `.githooks`
2. 记录自动发版配置：
   - `networkquiz.autoRelease=true`
   - `networkquiz.releaseRepo=ZhiKong0/review-baodian-apk`

当前默认配置已经切到现在线上真实存在并已发布历史版本的新仓库 `ZhiKong0/review-baodian-apk`。

## 行为

当你在这个仓库里完成一次 `git commit`，并且提交后工作区是干净的，`post-commit` hook 会自动执行：

1. 读取刚才这次修复提交的标题
2. 自动把版本号补成下一个补丁版本
3. 生成新的 `release/network_quiz_update.json`
4. 自动创建一条 `Release prep: vX.Y.Z` 提交
5. 自动推送分支和 tag
6. 自动发布 GitHub Release

## 跳过自动发版

如果某次提交你不想自动发版，在提交标题里加上：

```text
[skip-release]
```

例如：

```text
docs: tidy readme [skip-release]
```

## 日志

自动发版 hook 的运行日志会写到：

```text
build/out/auto_release_hook.log
```
